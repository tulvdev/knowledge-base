## Conventions
- One topic per file; keep each file small (aim: <= 200–300 lines).
- Prefer: checklists, “when to use”, “pitfalls”, small runnable snippets.
- Use headings consistently: **Context → Key ideas → Examples → Pitfalls → References**.
- File naming: `kebab-case.md`.

## How to add a new note
1. Create a new file under the closest topic folder.
2. Add it to the folder `README.md` index.
3. Include at least one example or snippet.
